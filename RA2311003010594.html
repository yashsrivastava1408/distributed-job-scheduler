1. Project Executive Summary

This project is a production-grade, distributed task queue and scheduler built using Node.js, TypeScript, PostgreSQL, and React. It features atomic queue claim locks to prevent double-execution, strict concurrency controls, multi-tenant workspace isolation, cron scheduling, and real-time observability.

2. Key Design & Reliability Decisions

  • Atomic Claiming (SKIP LOCKED): Claims jobs inside a single Postgres transaction using SELECT FOR UPDATE SKIP LOCKED. This guarantees that multiple worker instances running concurrently never claim the same task.
  • Tenant Workspace Isolation: Supports double authentication: JSON Web Tokens (JWT) to secure the browser dashboard, and SHA-256 hashed API Keys for fast programmatic submissions. Middleware enforces strict boundaries preventing horizontal privilege access.
  • Stale Worker Recovery (Reaper): Workers emit heartbeats every 5 seconds. A background Reaper daemon scans for workers offline for over 30 seconds, marks them offline, and safely requeues their active jobs back to the queue.
  • Cross-Process Event Bridge: Workers forward metrics and status updates over a REST bridge to the API process, which broadcasts live changes to the dashboard using Socket.IO.

4. Database Schema Structure

The PostgreSQL database consists of 12 fully normalized tables:

  • User / Organization / OrganizationMember / Project: Models multi-tenant boundaries.
  • Queue: Stores queue details, priorities, concurrency locks, and default retry policies.
  • Job: Tracks tasks, payloads, attempts, execution slots, and status.
  • ScheduledJob: Manages templates for recurring cron or delayed executions.
  • JobExecution / JobLog: Tracks runtime historical timelines and stdout streams.
  • Worker / WorkerHeartbeat: Telemetry metrics and active worker loads.
  • DeadLetterEntry: Dead letter entries for exhausted task retries.
  • RetryPolicy: Customizable retry strategies (exponential, linear, fixed delay).
